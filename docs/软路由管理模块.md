# 软路由管理模块 — 完整实现文档

## 概述

软路由管理模块为 SuniPIP 平台新增了"预配置工控机"销售与管理能力。客户购买工控机后，通过平台管理 WiFi 账号，每个 WiFi 账号绑定一个 SOCKS5 代理节点，实现**不同 WiFi 账号走不同代理**的效果。

### 核心流程

```
客户购买代理节点 → 激活路由器 → 创建 WiFi 账号 → 绑定代理节点
     ↓                                        ↓
FreeRadius 分配 VLAN → Clash 按 VLAN 子网路由 → 不同设备走不同代理
```

### 硬件

- 工控机: N2940, 4GB RAM, 32GB SSD, 4 网口
- 系统: Debian 12
- 软件栈: FreeRadius + Clash + WireGuard + dnsmasq + nftables

### 网络架构

```
Internet ← eth0 (WAN: DHCP/PPPoE)
               ↓
    ┌──────────────────────────────┐
    │      工控机 (Debian 12)       │
    │   FreeRadius + Clash + WG    │
    │   Go Agent + Nginx + dnsmasq │
    └──────────────────────────────┘
       ↓          ↓           ↓
    eth1        eth2         eth3
  172.10.0.1   (trunk)     100.64.1.1
  管理口       → AP          有线LAN
  隔离无网     WPA-EAP      直连不代理
```

## 一、新增数据库表

### WireGuard 管理隧道架构

WG 隧道**仅用于管理链路**（SSH 远程管理、配置下发），不参与流量路由。设备端配置使用 `Table = off`，不影响路由表。

```
            ┌─────────────────────┐
            │  WG 服务器 (境内主)   │  ← wg0, ListenPort=51820
            │  ssh_host, ssh_key  │
            └─────────┬───────────┘
                      │ wg set peer ... allowed-ips 10.10.0.x/32
    ┌─────────────────┼─────────────────┐
    │                 │                 │
  设备 A            设备 B            设备 C
  wg0: Table=off   wg0: Table=off   wg0: Table=off
  10.10.0.2/32     10.10.0.3/32     10.10.0.4/32

            ┌─────────────────────┐
            │  WG 服务器 (境外备)   │  ← wg0, ListenPort=51820
            │  ssh_host, ssh_key  │
            └─────────────────────┘
                (同样的 peer 配置)
```

- **设备注册时**：平台通过 SSH 自动将新 peer 添加到 WG 服务器（`wg set wg0 peer ... && wg-quick save wg0`）
- **管理后台**：可手动"同步 Peers"批量推送所有 peer 到 WG 服务器
- **WG 服务器表**：存储 SSH 连接信息（host/port/user/private_key）+ 角色（primary/backup）

共 8 个迁移文件（`backend/database/migrations/`）：

| 迁移文件 | 表名 | 用途 |
|---------|------|------|
| `000111_create_wg_servers_table` | `wg_servers` | WireGuard VPN 服务器（公钥/私钥/端点/网段） |
| `000112_create_router_devices_table` | `router_devices` | 设备库存：序列号、客户绑定、Agent 密钥、WG IP、配置版本、心跳状态 |
| `000113_create_router_wifi_accounts_table` | `router_wifi_accounts` | WiFi 账号：用户名/密码、VLAN ID、IP 前缀、代理订阅绑定 |
| `000114_create_router_device_wg_peers_table` | `router_device_wg_peers` | 设备 WireGuard 对等配置（每设备最多 2 个 WG 隧道） |
| `000115_create_router_config_snapshots_table` | `router_config_snapshots` | 配置版本快照（完整 JSON），用于审计和回滚 |
| `000116_create_router_event_logs_table` | `router_event_logs` | 设备事件日志：注册、绑定、配置变更、心跳等 |
| `000117_add_router_permissions` | `permissions` / `roles` | 6 个新权限，分配给管理员角色 |
| `000118_add_ssh_fields_to_wg_servers` | `wg_servers`（alter） | 添加 SSH 连接信息 + 服务器角色（primary/backup） |

### 运行迁移

```bash
sudo -u www php artisan migrate --force
```

## 六、Go Agent（sunipip-router-agent/）

运行在路由器上的守护进程，负责从平台拉取配置并应用到本地服务。

### 6.1 项目结构

```
sunipip-router-agent/
├── cmd/agent/main.go              # 入口：CLI 参数、生命周期管理
├── go.mod / go.sum                # Go 1.22, 依赖: yaml.v3 + x/crypto
├── internal/
│   ├── config/config.go           # 配置文件 /etc/sunipip/agent.json 读写
│   ├── api/
│   │   ├── client.go              # 平台 API 客户端（心跳/拉取/确认/上报）
│   │   └── register.go            # 首次注册流程（WG 密钥生成 + 令牌交换）
│   ├── services/
│   │   ├── freeradius.go          # 写 /etc/freeradius/3.0/mods-config/files/authorize
│   │   ├── clash.go               # 写 /etc/clash/config.yaml
│   │   ├── wireguard.go           # 写 /etc/wireguard/wg{0,1}.conf
│   │   ├── network.go             # VLAN 接口管理（ip link add/del）
│   │   ├── firewall.go            # 写 /etc/nftables.conf
│   │   ├── dhcp.go                # 写 /etc/dnsmasq.d/vlan-*.conf
│   │   └── helpers.go             # 原子文件写入工具
│   ├── manager/
│   │   ├── manager.go             # 配置编排：network → firewall → dhcp → wg → radius → clash
│   │   └── reconcile.go           # 每 5 分钟漂移检测 + 自动修复
│   ├── health/health.go           # 系统信息采集（CPU/内存/磁盘/WAN IP/ARP 表）
│   └── localapi/
│       ├── server.go              # HTTP 服务器（172.10.0.1:8080）
│       ├── handlers.go            # /api/status, /api/network, /api/services, /api/connected-devices, /api/restart-service
│       └── auth.go                # Bearer token 验证（缓存 5 分钟）
```

### 6.2 运行生命周期

1. **首次启动**：`install.sh` → 安装依赖 → 生成 WG 密钥 → POST /register → 写 agent.json
2. **主循环**（两个 goroutine）：
   - 心跳（60 秒）：上报系统信息 → 检测是否有新配置
   - 配置同步：config_version > applied → GET /config → 按序应用 → POST /ack-config
3. **漂移修复**（每 5 分钟）：检查服务运行状态和配置文件一致性，发现漂移自动重新应用
4. **本地 API**：为设备前端提供只读状态接口 + 服务重启能力

### 6.3 信号处理

| 信号 | 行为 |
|------|------|
| `SIGUSR1` | 立即触发配置拉取（通过 WG SSH 发送，延迟 ~2s） |
| `SIGTERM` / `SIGINT` | 优雅关闭 |

### 6.4 配置应用顺序

```
网络接口(VLAN) → 防火墙(nftables) → DHCP(dnsmasq) → WireGuard → FreeRadius → Clash
```

### 6.5 构建

```bash
cd sunipip-router-agent
go build -o sunipip-router-agent ./cmd/agent/
# 交叉编译（路由器是 amd64 Linux）：
GOOS=linux GOARCH=amd64 go build -o sunipip-router-agent ./cmd/agent/
```

## 七、安装脚本

### 7.1 install.sh — 一键安装

在裸机 Debian 12 工控机上执行：

```bash
INSTALL_TOKEN=xxx curl -fsSL https://admin.sunipip.uk/router-install/$INSTALL_TOKEN | bash
```

#### 安装流程

1. 检查环境（root、Debian 12、4 网口、读取序列号）
2. 安装依赖（freeradius, wireguard, dnsmasq, nftables, nginx, clash）
3. 生成 2 组 WireGuard 密钥对
4. 调用平台 `/register` API 注册设备
5. 写入 agent.json + WireGuard 配置
6. 配置网络接口（eth0 WAN / eth1 管理 / eth2 trunk / eth3 有线）
7. 配置 FreeRadius、dnsmasq、nftables
8. 安装 Clash + Go Agent（systemd 服务）
9. 部署设备前端 SPA + Nginx
10. 启用 IP 转发 + 启动所有服务

### 7.2 ap-config.sh — AP 配置

在 OpenWrt AP 上执行：

```bash
./ap-config.sh [路由器IP] [RADIUS密钥] [SSID名称]
```

配置 WPA2-Enterprise + 动态 VLAN，支持双频（2.4G + 5G）。

## 九、使用指南

### 9.1 管理员操作流程

#### 准备阶段

1. **创建 WG 服务器**：管理后台 → 软路由管理 → WG 服务器 → 添加（填入公网端点、密钥、网段）
2. **在 WG 服务器上部署**：下载配置文件 → 部署到实际 WireGuard 服务器

#### 设备入库

3. **添加设备**：软路由管理 → 设备库存 → 添加设备（输入序列号或留空）
4. **生成安装令牌**：点击设备 → 生成安装令牌 → 复制安装链接
5. **安装路由器**：在工控机上运行安装脚本

#### 客户绑定

### 9.2 客户操作流程

3. **连接使用**：手机/电脑连接 WiFi SSID → 输入 WiFi 账号密码 → 自动分配 VLAN → 通过绑定的代理上网
